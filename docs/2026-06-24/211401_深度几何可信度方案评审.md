# 深度几何可信度方案评审与优化建议

时间：2026-06-24 21:14:01

## 1. 审稿人判断

按 2027 TMI 审稿标准，当前 `mt_depth_guider_gan_v1` 和 `georisk_spc_dgv4_v2` 两条小实验还不足以支撑“深度几何可信度”这个核心主张。

主要原因不是结果一定差，而是方法 claim、代码实现、实验验证三者没有闭合：

- `georisk_spc_dgv4_v2` 当前更像 teacher entropy consistency，而不是 geometry trust。`_compute_risk_map_v2(depth, teacher_pred)` 虽然传入了 depth，但实际只用 teacher entropy 生成高/低风险区域，depth 没进入风险计算。
- `georisk_spc_dgv4_v2` 的 loss 主线也被削弱，只剩 supervised loss + clean/pert consistency；低风险 hard pseudo-label、边界一致性、风险区域监督没有形成完整闭环。
- `mt_depth_guider_gan_v1` 是 pred-depth 结构对抗约束，不是深度可信度估计。它无法回答“哪些 depth 几何信息可信，哪些不可信”。
- 当前训练和验证阶段仍输入 depth，因此不能声称“为纯 RGB 图像增加几何信息”。更准确的表述是 RGB + monocular pseudo-depth，或训练/测试都依赖预估深度。
- DepthGuiderV4 可以作为 depth feature injection 模块保留，但它本身不是可信度建模。它能支持“几何融合”，不能单独支持“深度可信度”。

结论：现在的主线如果直接投 TMI，审稿人会认为是 depth concat / depth-guided encoder / uncertainty filtering 的组合实验，可信度概念没有被清晰建模和验证。

## 2. 2026 文献压力

2026 年视角下，半监督医学分割和手术器械分割已经大量围绕 pseudo-label reliability 展开。单纯使用 entropy、confidence threshold、consistency regularization 不再有足够新意。

需要重点考虑的相关方向：

- `SegMatch: semi-supervised surgical instrument segmentation`，Scientific Reports 2025。该方法把 FixMatch 改到手术器械分割，并引入 adversarial augmentation，在 EndoVis 2017、Robust-MIS 2019、CholecInstanceSeg 上验证。
- `Endo-SemiS: Towards Robust Semi-supervised Image Segmentation for Endoscopic Video`，MIDL 2026。该方法使用 uncertainty-guided pseudo-label、dual-network cross-supervision、joint pseudo-label supervision 和 spatiotemporal corrective network。
- 2025-2026 医学 SSL 中已经大量出现 uncertainty-guided pseudo-label refinement、region learning、foundation-model pseudo-label refinement、cross-prompt consistency、pseudo-label correction 等设计。
- `Depth Anything V2`，NeurIPS 2024，让 monocular depth foundation model 更容易作为外部 dense cue 使用。
- RGB-D/RGB-pseudo-depth 分割方向已有 DFormer/DFormerv2、PrimKD、PDDM 等方法，说明“使用 depth/pseudo-depth 提升分割”本身不是强 novelty。

因此你的差异点不能停留在“用 depth”或“用 uncertainty”，必须落在：

> 单目深度来自 foundation model，不能默认可信；我们显式估计其局部几何可信度，并用该可信度决定未标注像素的监督方式。

## 3. 推荐新主线

建议把论文主线改为：

**Geometry-Trust Semi-Supervised Learning for RGB Surgical Instrument Segmentation**

核心定位：

- 输入场景是 RGB surgical image。
- monocular depth 只作为训练阶段的 privileged geometry cue。
- 方法显式估计 depth geometry trust map。
- 可信 depth 区域用于加强几何边界和伪标签监督。
- 不可信 depth 区域避免 hard pseudo-label，改用 soft consistency 或降低权重。
- 最终 student 在测试阶段只输入 RGB。

这条主线比当前 RGB-D 方案更容易被 TMI 接受，因为它解决的是实际问题：临床/测试部署时只有 RGB，单目深度来自大模型且质量不稳定，不能无脑融合。

## 4. 建议方法结构

### 4.1 Depth Reliability Map

定义一个像素级或区域级 `Q_d`，表示 depth geometry 是否可信。

建议由三类信号组成：

1. Depth stability：同一 RGB 在弱增强、多尺度、flip 或不同 depth estimator 下得到的 depth/gradient 是否一致。
2. RGB-depth edge agreement：RGB edge 与 depth discontinuity 是否空间对齐。
3. Geometry-semantic conflict：depth boundary 与 teacher semantic boundary 是否冲突。

可以简化成：

```text
Q_d = exp(-Var(G_d)) * Align(E_rgb, G_d) * (1 - Conflict(G_d, B_t))
```

其中：

- `G_d` 是 depth gradient / depth discontinuity。
- `E_rgb` 是 RGB edge 或 image gradient。
- `B_t` 是 teacher prediction boundary。
- `Conflict` 衡量 depth boundary 与 semantic boundary 的不一致。

重点不是公式复杂，而是 `Q_d` 必须能被审计：可信区域 pseudo-label 是否真的更准，不可信区域是否错误更多。

### 4.2 Region-Aware Supervision

未标注像素不要只按 teacher confidence 分流，而是按 confidence 和 depth trust 联合分流：

- `high confidence + high Q_d`：使用 hard pseudo-label CE。
- `high confidence + low Q_d`：降低 pseudo-label 权重，避免错误 depth 引导。
- `low confidence + high Q_d`：使用 geometry boundary consistency 或 soft consistency。
- `low confidence + low Q_d`：只做弱 consistency 或忽略。

这样才能体现“深度不是永远可信，可信时用，不可信时抑制”。

### 4.3 RGB-Only Student

建议新增或改造为 train-depth/test-RGB：

- teacher/risk module 可以看到 RGB + depth。
- student 主分割分支只输入 RGB。
- depth 通过 loss、distillation 或 feature guidance 影响训练，不作为测试输入。

这能把论文从“RGB-D 分割”提升为“利用单目深度先验增强 RGB 分割”。

### 4.4 DepthGuiderV4 的位置

DepthGuiderV4 可以保留为辅助实验：

- `RGB baseline`
- `RGB + depth concat`
- `DepthGuiderV4`
- `DepthGuiderV4 + GeoTrust`
- `GeoTrust train-depth/test-RGB`

但不建议把 DepthGuiderV4 作为核心创新。它更适合作为几何融合模块或 upper-bound/variant。

### 4.5 不建议继续主推 GAN

`mt_depth_guider_gan_v1` 不建议作为主线。

原因：

- GAN 判别的是 pred-depth 结构，不是 depth reliability。
- 对抗训练稳定性差，审稿人会要求大量稳定性分析。
- 它无法解释单目 depth 错误时为什么不会误导模型。
- 当前 v1 已经存在未稳定伪标签过早参与 GAN 的设计问题。

GAN 最多作为 supplementary exploratory result，不应作为 TMI 主贡献。

## 5. 必须补充的实验

### 5.1 主结果

至少 Task 1 做完整 4-fold：

- label ratio：5%、10%、20%、40%
- baselines：Fully、MT、UAMT、URPC、CPS、UniMatch、U2PL、SegMatch
- metrics：Dice、IoU、HD95、ASD、Boundary F1

如果主张泛化到多类别，再补 Task 2/Task 3，但不要让多类别拖垮主线。

### 5.2 RGB-only 对比

必须单独成表：

- RGB student
- RGB + pseudo-depth concat
- DepthGuiderV4
- train-depth/test-RGB distillation
- proposed GeoTrust

如果 proposed 仍然测试输入 depth，就不能写纯 RGB。

### 5.3 Depth Trust 审计

这是最关键实验：

- `Q_d` 对 pseudo-label error 的 AUROC/AUPRC。
- low-trust vs high-trust 区域 pseudo-label accuracy。
- risk coverage vs pseudo-label error rate。
- high depth-gradient band 内的 Dice/Boundary F1。
- 可视化：RGB、depth、depth gradient、`Q_d`、teacher error、final prediction。

审稿人需要看到：`Q_d` 真能定位 depth/伪标签不可靠区域，而不是一个好看的 heatmap。

### 5.4 Depth 破坏实验

证明你的方法不会盲信 depth：

- depth blur
- depth noise
- depth shuffle
- random depth
- depth edge erosion/dilation
- 不同 depth estimator：DPT、Depth Anything V2、Metric3D

理想结论：

- 普通 RGB-D concat 在坏 depth 下明显下降。
- GeoTrust 在坏 depth 下自动降低 depth supervision 权重，下降更小。

### 5.5 消融实验

建议消融：

- no `Q_d`
- entropy-only
- depth-edge-only
- RGB-depth alignment only
- semantic-depth conflict only
- no hard pseudo-label gating
- no geometry boundary consistency
- train-depth/test-RGB vs train-test-depth

如果某些消融超过 full，不要强行解释为 full 最优。应该承认固定阈值/权重带来的校准问题，并改成 adaptive threshold 或 percentile selection。

## 6. 当前代码的直接改进方向

优先级最高：

1. 不要继续扩展 `georisk_spc_dgv4_v2` 当前版本。它已经偏离 geometry trust 主张。
2. 回到 `semi_georisk_spc.py` 的完整风险监督逻辑，但把风险图从 `R = U_t * G_d + conflict` 升级为可信度 `Q_d`。
3. 新增一个明确实验名，例如 `geotrust_rgb_v1`，避免继续在 `georisk_spc_dgv4_v2` 上堆概念。
4. 保证 loss 返回包含 `total`、`sup`、`pl_high_trust`、`cons_low_trust`、`bd_geo`、`trust_ratio`、`depth_trust_mean`。
5. 测试阶段如果要讲 RGB-only，`validation_step` 和 `core.test` 必须只喂 RGB。

## 7. 建议论文贡献表述

可以写成：

1. We formulate monocular-depth-assisted surgical instrument segmentation as a geometry-trust problem rather than direct RGB-D fusion.
2. We propose a pixel-level depth geometry trust estimator that combines depth stability, RGB-depth edge agreement, and geometry-semantic conflict.
3. We design trust-conditioned unlabeled supervision, applying hard pseudo-labels only when both semantic confidence and depth geometry are reliable, while using soft consistency in unreliable regions.
4. We train an RGB-only student with depth as privileged training geometry, improving deployment compatibility.
5. We provide explicit depth-trust audits and corruption studies showing that the method benefits from reliable depth while resisting unreliable monocular depth.

## 8. 最终建议

当前方案不建议直接包装成 TMI 主线。最稳妥的路线是放弃 GAN 主线，重构为“单目深度可信度驱动的 RGB-only 半监督器械分割”。

核心判断：

> 不要证明 depth 有用；这个已经有大量文献。要证明 depth 什么时候可信、什么时候不可信，以及你的训练策略能根据可信度选择正确监督信号。
