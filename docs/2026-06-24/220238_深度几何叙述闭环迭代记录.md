# 深度几何叙述闭环迭代记录

时间：2026-06-24 22:02:38

## 0. 目标

围绕“用单目深度几何信息增强 RGB 手术器械分割”这个方向，反复检查叙述是否闭环，尤其判断：

- 为什么需要 depth。
- 为什么 depth 不能直接用。
- 为什么需要可信度或隐式仲裁。
- 为什么不是重复已有多模态可靠性工作。
- 最终还剩下什么值得作为 TMI 级别问题。

## 1. V0：原始叙述

原始想法：

> 更好地使用深度几何信息，为纯 RGB 图像增加几何信息；由于 depth 来自单目深度估计大模型，不能完全信任，因此提出深度几何可信度。

### 问题

这个叙述动机是通的，但没有形成方法闭环。

断点：

- “纯 RGB”与当前代码不一致，当前训练/验证仍输入 depth。
- “深度不可信”只出现在动机里，方法没有真正建模 depth 何时不可信。
- “可信度”没有 ground truth，显式建模容易变成 heuristic。
- 结果如果只看 Dice，无法证明可信度机制有因果作用。

### 结论

V0 不能作为论文主线。

它只是一个方向描述，不是可验证科学问题。

## 2. V1：显式可信度闭环

尝试叙述：

> RGB-only 手术器械分割缺少几何先验 → 单目深度大模型可提供几何提示但不稳定 → 直接融合会被坏 depth 误导 → 提出深度几何可信度，决定哪些 depth 区域可用 → 通过 clean/corrupted depth 和伪标签错误审计证明有效。

### 优点

- 开始区分“depth 有用”和“depth 不总可信”。
- 引入了 clean/corrupted depth 这种因果验证。
- 明确了直接 concat/DepthGuider 的风险。

### 问题

显式可信度仍然不够强。

断点：

- `trust map = entropy + depth edge + conflict` 显得手工。
- 可信度到底是 depth accuracy、pseudo-label reliability、boundary usefulness，定义不清。
- 没有 depth trust ground truth，评估只能间接做。
- 2024-2026 已有大量 uncertainty-guided pseudo-label 和 multimodal reliability 方法，显式可信度不是强 novelty。

### 结论

“可信度”可以作为动机，但不适合作为方法核心。

## 3. V2：隐式几何仲裁闭环

调整叙述：

> 不是显式预测 depth 是否可信，而是训练一个隐式几何仲裁机制，学习何时使用或抑制 depth-derived geometry。

具体路径：

> RGB-only 手术器械分割缺少几何先验 → 单目 depth 提供 noisy geometry → 直接 depth fusion 会被坏 geometry 误导 → 隐式 geometry arbitration 学习几何使用强度 → 测试只用 RGB → 通过 depth corruption、RGB-only inference、边界指标和 pseudo-label error audit 验证。

### 优点

- 避免显式可信度标签缺失问题。
- 方法从“手工可信度”转向“任务驱动几何使用”。
- 可以用 gate/router/residual arbitration 表达，不必声明 gate 是准确可信度。

### 问题

这一版仍然不够闭环。

断点：

- 仍没有充分解释“为什么必须用 depth，而不是 SAM、optical flow、DINO feature、temporal cue、edge map”。
- “几何仲裁”本身和多模态可靠性融合非常相似。
- 如果测试仍用 depth，就是 RGB-D reliability fusion，不新。
- 如果只是 clean/corrupted depth 鲁棒性，也容易被归入已有 unreliable modality fusion。

### 结论

V2 比 V1 更好，但仍可能被审稿人认为是多模态可靠性工作的迁移。

## 4. V3：因果必要性闭环

进一步具体化每一步因果必要性：

### 4.1 为什么要几何

手术器械分割的关键错误集中在结构区域：

- 细长器械断裂。
- 器械边界泄漏。
- 遮挡处前景/背景混淆。
- 反光导致局部纹理误导。
- 器械-组织接触处边界不清。

这些错误不是普通类别识别错误，而是几何结构错误，需要边界、前后关系、局部表面突变等信息。

### 4.2 为什么用 depth 表达几何

单帧 RGB 中几何关系是隐式的。

monocular depth 能显式提供：

- depth discontinuity。
- occlusion cue。
- tool-tissue separation。
- 相对前后关系。
- 结构边界提示。

这里需要的不是准确绝对深度，而是对分割有用的相对几何提示。

### 4.3 为什么不能直接用 depth

单目 depth 是外部大模型估计，不是手术场景真值。

它在以下区域容易错：

- 反光。
- 烟雾。
- 血液。
- 运动模糊。
- 细器械。
- 器械与组织颜色接近区域。

直接 concat 或 attention 默认 depth 总有益，错误 depth 会进入 feature，污染边界预测。

### 4.4 为什么这是半监督问题

在 unlabeled data 上，没有 GT 能纠正错误 depth 和错误 pseudo-label。

teacher 如果在边界错且 confidence 高，hard pseudo-label 会强化错误。

坏 depth 进一步稳定这种错误边界，形成 confirmation bias。

### 4.5 真正问题

不是：

> 如何使用 depth？

而是：

> 在没有 depth 真值、没有 unlabeled GT 的情况下，如何从 noisy pseudo-depth 中只提取对 RGB 分割有益的几何，同时避免错误几何污染伪标签训练？

### 4.6 为什么隐式仲裁

depth 是否“可信”没有直接监督标签。

显式 trust map 只能依赖手工假设。

更合理的是让模型根据：

- segmentation objective。
- teacher-student disagreement。
- RGB-depth geometry consistency。
- pseudo-label stability。
- boundary consistency。

学习 depth-derived geometry 的使用强度。

### 4.7 为什么 RGB-only 测试

如果测试输入 depth，论文就是 RGB-D/pseudo-depth segmentation。

如果测试只输入 RGB，depth 才是 privileged training information，论文才真正回答“如何为纯 RGB 图像增加几何信息”。

### 结论

V3 的逻辑比 V2 更完整，但仍有一个外部威胁：多模态可靠性和 RGB-D surgical segmentation 相关工作已经很接近。

## 5. V4：与已有工作的差异化闭环

检查相关工作后，需要承认：

> “多模态信息不可靠，所以做可靠性融合”这个问题已经不新。

已有相近方向包括：

- 多模态 reliability fusion：例如 conflict-guided/evidential multimodal fusion、reliability-aware fusion。
- RGB-D semantic segmentation：已有大量 depth fusion、primary modality、cross-modal interaction 方法。
- 手术场景 depth/RGB-D：已有 surgical RGB-D segmentation 和 depth for surgical foundation models。
- 半监督器械分割：已有 SegMatch、Endo-SemiS 等 pseudo-label/uncertainty/robust SSL 方法。

因此，如果只说：

> depth 不可靠，所以做可靠性。

这个方向重复，不够有意义。

### 最终可成立的窄问题

必须同时满足四个限定：

1. 测试时只有 RGB。
2. 训练时可获得 monocular pseudo-depth。
3. pseudo-depth 来自 foundation model，既有用又有任务错配噪声。
4. 半监督训练中错误 depth 会与错误伪标签相互强化。

最终问题应写成：

> Existing multimodal reliability methods address reliable fusion when multiple modalities are available at inference. We address a different setting: only RGB is available at inference, while noisy monocular pseudo-depth is available only during training. The challenge is to distill useful geometry into an RGB-only semi-supervised student without letting erroneous pseudo-depth amplify pseudo-label confirmation bias.

中文版本：

> 已有多模态可靠性方法主要解决测试时多模态可用情况下如何可靠融合；我们解决的是测试时只有 RGB、训练时只有 noisy pseudo-depth 的半监督器械分割问题：如何把有用几何蒸馏进 RGB-only student，同时防止错误 pseudo-depth 放大伪标签 confirmation bias。

## 6. 最终闭环版本

最终推荐叙述：

> 手术器械 RGB-only 分割的主要错误集中在几何结构区域，如细器械边界、遮挡、反光和器械-组织接触处。单目深度大模型能从 RGB 中提供相对几何提示，如深度突变和前后关系，但这些 pseudo-depth 在手术场景中存在任务错配和局部错误。直接把 pseudo-depth 作为第四通道或 depth-guided feature 输入，会在坏 depth 区域引入错误几何；在半监督训练中，这些错误几何还会与 teacher pseudo-label 错误相互强化，造成 confirmation bias。不同于已有测试时多模态可靠性融合，我们研究测试时只有 RGB 的设置，将 noisy pseudo-depth 作为训练时 privileged geometry。为此提出隐式几何仲裁机制，在无 depth 真值和无 unlabeled GT 的情况下，学习何时吸收或抑制 depth-derived geometry，并将有用几何蒸馏到 RGB-only student。通过 clean/corrupted depth、RGB-only inference、边界指标和 pseudo-label error audit 证明：几何确实有用，直接融合会受坏 depth 影响，隐式仲裁能抑制错误几何，最终 RGB-only student 仍能获益。

## 7. 一句话版本

> 我们不是做测试时 RGB-D 可靠融合，而是把 noisy monocular depth 作为训练时 privileged geometry，用隐式仲裁选择性吸收其对半监督器械分割有用的结构信息，并防止错误 depth 放大伪标签错误，最终提升 RGB-only 推理。

## 8. 这个闭环成立的必要实验

如果缺少这些实验，闭环仍然不成立：

- RGB-only inference：证明测试不依赖 depth。
- Depth corruption curve：证明不是盲信 depth。
- Direct fusion baseline：证明简单 RGB-D/DepthGuider 会被坏 depth 误导。
- Clean depth gain：证明 depth-derived geometry 确实有用。
- Pseudo-label error audit：证明半监督错误传播被抑制。
- Boundary metric：证明收益发生在几何结构区域。
- 与 SegMatch/Endo-SemiS/UniMatch/U2PL 等强 SSL baseline 对比：证明不是旧 SSL 方法即可达到。

## 9. 当前最重要结论

如果只做“depth 可信度”或“RGB-D 可靠融合”，方向重复。

真正还有价值的是这个窄问题：

> Noisy pseudo-depth as privileged training geometry for RGB-only semi-supervised surgical instrument segmentation.

这个问题比“深度可信度”更稳，也更容易和已有多模态可靠性方法拉开距离。
