# 反事实几何仲裁方案

时间：2026-06-24 22:47:15

## 1. 方案定位

建议方向：

**反事实几何仲裁的 RGB-only 半监督手术器械分割**

英文可写：

**Counterfactual Geometry Arbitration for RGB-only Semi-supervised Surgical Instrument Segmentation**

核心思想：

> 做 RDNet 的反问题。

RDNet 的主张是：

> depth can help RGB semi-supervised learning.

我们的主张应改成：

> noisy pseudo-depth can both help and harm RGB semi-supervised learning; in surgical scenes, wrong geometry may corrupt pseudo-label learning, so depth-derived geometry must be selectively allowed or suppressed.

也就是说，不再重复“depth 帮助 RGB SSL”，而是研究“什么时候 depth 不应该帮助 RGB SSL”。

## 2. 为什么需要这个方案

2025 CVPR RDNet 已经做了训练时 depth 辅助、测试 RGB-only 的半监督闭环：

- RGB student 最终用于推理。
- depth student 只在训练时辅助。
- depth branch 高置信预测用于更新 RGB pseudo-label。
- DPA 使用 depth variance 做 hard patch augmentation。

因此，如果继续做“depth 辅助 RGB-only SSL”，容易重复 RDNet。

新的切入点必须是：

> RDNet 默认 depth guidance 是有益的；但 surgical pseudo-depth 来自单目深度大模型，在反光、细器械、遮挡、血液、模糊区域可能产生错误几何。错误 depth guidance 不仅不能帮助 RGB，还可能错误修正 RGB pseudo-label，放大半监督 confirmation bias。

这才是更具体、更有差异的问题。

## 3. 与 RDNet 的关键区别

| 维度 | RDNet | 本方案 |
|---|---|---|
| 核心假设 | depth branch 可辅助 RGB branch | pseudo-depth 有时辅助，有时误导 |
| depth 使用 | 高置信 depth prediction 更新 RGB pseudo-label | 只有通过反事实稳定性检验的 geometry 才能参与更新 |
| 风险建模 | 主要依赖 depth confidence / variance | 显式研究坏 depth 对 SSL 伪标签的污染 |
| 训练目标 | depth-guided co-training | counterfactual geometry arbitration |
| 测试输入 | RGB-only | RGB-only |
| 核心实验 | 半监督性能提升 | clean/corrupted depth 下的收益与抗干扰 |
| 适用场景 | polyp segmentation | surgical instrument segmentation |

一句话差异：

> RDNet asks how depth can help RGB SSL; we ask when pseudo-depth should not be trusted to help RGB SSL.

## 4. 方法总览

保留 RGB-only student 和训练时 geometry assistant。

训练阶段：

1. RGB teacher/student 对 unlabeled RGB 产生 pseudo-label。
2. geometry assistant 使用 pseudo-depth 产生 geometry-aware prediction。
3. 对同一张 RGB 构造多个 counterfactual depth views。
4. 检查 geometry assistant 在这些 depth views 下是否稳定。
5. 根据稳定性、RGB confidence、RGB-geometry disagreement 生成 geometry arbitration weight。
6. arbitration weight 直接控制 unlabeled supervision 的形式和强度。

测试阶段：

- 只保留 RGB student。
- 不输入 depth。

## 5. 模块设计

### 5.1 RGB Student / Teacher

主分支是 RGB-only。

输入：

```text
x_u
```

输出：

```text
p_rgb = T_rgb(x_u)
```

其中 `T_rgb` 是 EMA teacher，`p_rgb` 是 soft pseudo-label。

### 5.2 Geometry Assistant

辅助分支训练时使用 pseudo-depth。

输入可以是：

```text
[x_u, d_u]
```

或者只用：

```text
d_u
```

输出：

```text
p_geo = G(x_u, d_u)
```

这个分支不用于最终推理。

### 5.3 Counterfactual Depth Views

对同一张未标注图像构造多个 depth view：

```text
d_clean
d_blur
d_noise
d_edge_drop
d_shuffle_patch
```

不需要太多，建议先从三种开始：

- clean depth
- blurred depth
- edge-dropped depth

得到：

```text
p_geo_clean = G(x_u, d_clean)
p_geo_cf = G(x_u, d_cf)
```

### 5.4 Geometry Stability

用 geometry assistant 在 clean/counterfactual depth 下的预测差异定义稳定性。

示例：

```text
S_geo = exp(-KL(p_geo_clean || p_geo_cf))
```

或者：

```text
S_geo = 1 - mean(|p_geo_clean - p_geo_cf|)
```

这里 `S_geo` 不叫可信度，不作为单独监督目标。

推荐名称：

- geometry stability
- counterfactual geometry stability
- geometry arbitration evidence

### 5.5 RGB-Geometry Disagreement

定义 RGB teacher 与 geometry assistant 的分歧：

```text
D_rg = KL(p_rgb || p_geo_clean)
```

分歧大说明 geometry guidance 可能会改变 RGB pseudo-label，需要谨慎。

### 5.6 Geometry Arbitration Weight

根据以下信号生成连续权重：

- RGB teacher confidence
- geometry stability `S_geo`
- RGB-geometry disagreement `D_rg`
- 可选：depth boundary strength

简单版本：

```text
A = S_geo * conf_geo * exp(-D_rg)
```

或者学习一个小 gate：

```text
A = Gate(f_rgb, f_geo, S_geo, D_rg)
```

注意：

- `A` 不叫 trust map。
- `A` 不需要 ground truth。
- `A` 是 unlabeled supervision arbitration weight。

## 6. Unlabeled Supervision 设计

这是方案闭环的核心。

### 6.1 高稳定几何区域

条件：

```text
RGB confidence high
geometry stability high
RGB-geometry disagreement low
```

监督：

- 允许 hard pseudo-label。
- 可融合 RGB teacher 和 geometry assistant。

```text
y_hat = argmax((1 - A) * p_rgb + A * p_geo)
L_pl = CE(p_student, y_hat)
```

### 6.2 几何冲突区域

条件：

```text
RGB confidence high
geometry stability low or RGB-geometry disagreement high
```

监督：

- 降低 hard pseudo-label 权重。
- 避免 geometry assistant 错误修正 RGB。

```text
L_pl = (1 - A) * CE(p_student, argmax(p_rgb))
```

### 6.3 RGB 不确定但几何稳定区域

条件：

```text
RGB confidence low
geometry stability high
depth/geometry boundary clear
```

监督：

- 不直接 hard label。
- 使用 soft consistency 或 boundary consistency。

```text
L_cons = M_geo * MSE(p_student, p_geo.detach())
L_bd = M_geo * |grad(p_student) - grad(p_geo.detach())|
```

### 6.4 双低稳定区域

条件：

```text
RGB confidence low
geometry stability low
```

监督：

- 忽略或极低权重。

## 7. Loss 主线

整体 loss：

```text
L = L_sup_rgb
  + lambda_geo_sup * L_sup_geo
  + lambda_pl * L_arbitrated_pl
  + lambda_cons * L_arbitrated_cons
  + lambda_bd * L_arbitrated_bd
  + lambda_align * L_rgb_geo_align
```

其中关键不是 loss 数量，而是：

> geometry arbitration 直接控制 unlabeled loss。

如果 geometry 只进入 feature fusion，而不控制 unlabeled supervision，就又退化成普通 RGB-D/DepthGuider。

## 8. 实验设计

### 8.1 主对比

必须包含：

- RGB Mean Teacher
- UniMatch / U2PL / SegMatch
- RDNet-style depth update
- RGB + depth concat
- DepthGuiderV4
- Ours without counterfactual arbitration
- Ours full

### 8.2 Clean / Corrupted Depth Curve

这是最关键实验。

depth 设置：

- clean depth
- blur depth
- noise depth
- edge-drop depth
- patch-shuffle depth
- random depth

预期：

- clean depth 下，ours 优于 RGB baseline。
- corrupted depth 下，ours 下降小于 RDNet-style update、RGB-D concat、DepthGuiderV4。
- random depth 下，ours 接近 RGB baseline，不应明显崩溃。

### 8.3 RGB-only Inference

必须证明最终推理只输入 RGB。

对比：

- train RGB / test RGB
- train RGB+depth / test RGB+depth
- train RGB+depth / test RGB
- proposed train with geometry arbitration / test RGB

### 8.4 Pseudo-label Error Audit

在 held-out labeled set 上做 post-hoc audit：

- 被 `A` 降权的区域 pseudo-label error 是否更高。
- 被 `A` 允许 geometry 更新的区域 pseudo-label 是否更准。
- depth corruption 后 `A` 是否下降。

不要声称 `A` 是真实可信度，只分析它是否与伪标签错误传播相关。

### 8.5 Boundary Metrics

报告：

- Boundary F1
- HD95
- ASD
- high-gradient band Dice / IoU

因为方法动机是几何结构区域，不能只报整体 Dice。

## 9. 论文贡献写法

建议写：

1. We revisit depth-assisted RGB-only semi-supervised segmentation from the perspective of harmful pseudo-depth guidance.
2. We propose counterfactual geometry arbitration, which measures whether depth-derived guidance remains stable under depth perturbations before allowing it to influence unlabeled supervision.
3. We use geometry arbitration to control pseudo-label updating, soft consistency, and boundary consistency, preventing erroneous geometry from amplifying confirmation bias.
4. The final model is RGB-only at inference, using pseudo-depth only as privileged geometry during training.
5. Extensive clean/corrupted depth experiments and pseudo-label error audits show that the method benefits from useful geometry while suppressing harmful geometry.

## 10. 不建议的表述

避免写：

- We estimate depth reliability.
- We compute a trustworthy depth map.
- Depth confidence is accurate.
- Geometry trust is supervised.

推荐写：

- We learn when depth-derived geometry should influence unlabeled supervision.
- We use counterfactual depth perturbations to detect unstable geometry guidance.
- Geometry arbitration controls pseudo-label usage rather than serving as an explicit reliability target.

## 11. 最小可实现版本

为了先验证方向，建议最小实现：

1. 复用 RDNet 的 RGB branch + depth branch。
2. 对 depth branch 增加 depth perturbation forward。
3. 用 `|p_geo_clean - p_geo_pert|` 计算 `S_geo`。
4. 用 `S_geo` 和 RGB-depth disagreement 替代 RDNet 里直接按 depth confidence 更新 pseudo-label。
5. 保持最终 validation/test 只用 RGB branch。

最小 ablation：

- RDNet-style update。
- RDNet + counterfactual stability gate。
- RDNet + random/corrupted depth。

如果这三个实验没有明显差异，方向不成立。

## 12. 最终一句话

> 我们不是简单用 depth 帮助 RGB，也不是普通多模态可靠融合；我们研究 noisy pseudo-depth 在半监督手术器械分割中何时会错误修正伪标签，并用反事实几何稳定性仲裁 unlabeled supervision，最终训练出 RGB-only 分割模型。
