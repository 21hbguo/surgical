# 冲击 TMI 高分方案升级

时间：2026-06-24 22:54:23

## 1. 当前方案评分

当前“反事实几何仲裁”方案如果投 TMI，大约是：

```text
6/10
```

原因：

- RDNet 已经覆盖了 depth 辅助 RGB-only 半监督。
- 多模态可靠性已经覆盖了 unreliable modality fusion。
- counterfactual depth perturbation 容易被认为是 robustness trick。
- 如果只做 blur/noise/drop depth，很难体现医学和手术场景特异性。

想达到：

```text
9.5/10
```

必须把方案从“depth perturbation gate”升级成：

> 手术视频中 noisy pseudo-depth 的时序-因果几何验证。

## 2. 核心升级方向

建议新主线：

**Temporal-Causal Geometry Arbitration for RGB-only Semi-supervised Surgical Instrument Segmentation**

中文：

**面向 RGB-only 半监督手术器械分割的时序-因果几何仲裁**

核心变化：

从：

> depth perturb 后预测是否稳定。

升级为：

> depth-derived geometry 是否对当前未标注区域的结构监督产生正向因果贡献。

这比单纯可信度、单纯可靠性、单纯 depth corruption 更强。

## 3. 为什么需要升级

### 3.1 RDNet 已经做了 depth helps RGB SSL

RDNet 的闭环：

```text
RGB SSL 伪标签不够好
→ depth branch 提供辅助
→ depth 高置信预测更新 RGB pseudo-label
→ DPA 用 depth variance 找 hard patches
→ 测试 RGB-only
```

如果我们的方案只是：

```text
用 depth 辅助 RGB SSL，测试 RGB-only
```

就重复 RDNet。

### 3.2 多模态可靠性已经很多

如果只说：

```text
depth 不可靠，所以做 reliability/gating
```

也不够，因为已有多模态可靠性、conflict-guided fusion、evidential fusion、sensor degradation fusion 等工作。

### 3.3 必须引入手术视频特异性

手术器械分割的关键特殊性：

- 细器械。
- 反光。
- 血液和烟雾。
- 器械-组织接触。
- 遮挡。
- 视频中器械和相机运动导致 depth 抖动。

这些不是普通 RGB-D fusion 的核心问题。

因此，高分方案必须利用：

- 手术视频时序。
- depth 几何的局部错误。
- 半监督伪标签错误传播。
- RGB-only 推理部署需求。

## 4. 9.5 分版本核心思想

核心句：

> We do not use pseudo-depth to correct semantic labels directly. Instead, we causally validate whether depth-derived geometry provides stable structural evidence across RGB/depth counterfactuals and temporal neighbors, and only then use it to arbitrate unlabeled structural supervision for an RGB-only student.

中文：

> 我们不让 pseudo-depth 直接修正语义伪标签，而是通过 RGB/depth 双反事实和时序邻域验证 depth-derived geometry 是否提供稳定结构证据；只有通过验证的几何才参与未标注结构监督仲裁，最终训练 RGB-only student。

这个表述比“深度可信度”强很多。

## 5. 三个关键方法升级

### 5.1 双视角反事实验证

当前方案只扰动 depth：

```text
d_clean -> p_geo_clean
d_pert  -> p_geo_pert
```

升级后同时扰动 RGB 和 depth：

```text
x_clean, d_clean
x_pert,  d_clean
x_clean, d_pert
x_pert,  d_pert
```

目的：

- 如果 depth 轻微扰动导致 prediction 大变，说明 geometry guidance 不稳定。
- 如果 RGB appearance 扰动后 depth geometry 仍提供稳定 boundary，说明 geometry 是有用结构锚点。
- 如果 RGB 和 depth 都变时预测仍稳定，说明该区域监督更可靠。

可定义：

```text
S_depth = sim(G(x, d_clean), G(x, d_pert))
S_rgb = sim(G(x_clean, d), G(x_pert, d))
CausalGeo = S_depth * S_rgb * exp(-Disagree(p_rgb, p_geo))
```

这里不要叫 trust map。

建议命名：

- causal geometry evidence
- geometry validation score
- temporal-causal arbitration weight

### 5.2 depth 只纠正结构，不纠正语义

RDNet 里 depth branch 可以更新 RGB pseudo-label。

这在 surgical pseudo-depth 下有风险，因为 depth 错误可能直接改错前景/背景。

高分方案应限制 depth 的作用：

- depth 不直接决定类别。
- depth 只影响 boundary、shape、region continuity。
- semantic class 仍由 RGB teacher/student 决定。

具体做法：

- hard semantic pseudo-label 来自 RGB teacher。
- depth-derived geometry 只决定 pseudo-label weight、boundary consistency、shape consistency。
- 如果 geometry 与 RGB 语义冲突，只降低权重，不直接改 label。

这样更符合医学审稿人的直觉：

> 几何不能替代语义，但可以约束结构。

### 5.3 加入时序一致性

手术是视频任务。

错误 monocular depth 往往有时序抖动；真实器械结构边界更具有局部时序连续性。

利用相邻帧：

```text
x_t, x_{t-1}, x_{t+1}
d_t, d_{t-1}, d_{t+1}
```

可计算：

- depth boundary temporal consistency。
- geometry prompt temporal stability。
- pseudo-label boundary temporal stability。

不需要做复杂 optical flow，最小版本可以用邻近采样和弱对齐：

- 同一 sequence 邻近 frame。
- resize 后直接比较低频 boundary map。
- 或只比较全局/patch-level stability。

时序模块的意义：

- 明显区别于 RDNet。
- 明显体现 surgical video 特性。
- 能解释为什么手术场景需要这个方法。

## 6. 升级后完整闭环

最终闭环：

1. 低标注手术视频需要半监督学习。
2. 半监督器械分割的错误集中在边界、遮挡、反光和细长结构区域。
3. 这些区域的错误本质上是结构/几何错误，而不是单纯语义错误。
4. 单目深度大模型能提供 depth discontinuity 和相对几何提示。
5. 但 surgical pseudo-depth 在反光、血液、烟雾、细器械处存在局部错误和时序抖动。
6. 直接 depth co-training 会把错误几何注入未标注监督，放大 pseudo-label confirmation bias。
7. 因此，depth 不应直接修正语义 pseudo-label，而应先通过 RGB-depth 双反事实和时序一致性进行几何因果验证。
8. 通过验证的几何只用于结构监督：boundary consistency、shape consistency、pseudo-label weight arbitration。
9. 最终 student 测试只输入 RGB。
10. 通过 clean/corrupted depth、RGB/depth counterfactual、temporal jitter audit、RGB-only inference 证明方法有效。

一句话：

> 用 noisy pseudo-depth 不是为了替代 RGB 语义，而是为了在半监督训练中提供经过时序-因果验证的结构监督，最终提升 RGB-only 手术器械分割。

## 7. 方法框架

### 7.1 RGB 主分支

输入：

```text
x_t
```

输出：

```text
p_rgb_t
```

RGB 分支是最终推理模型。

### 7.2 Geometry Assistant

输入：

```text
x_t, d_t
```

输出：

```text
g_t
b_geo_t
p_geo_t
```

其中：

- `g_t` 是几何 feature/prompt。
- `b_geo_t` 是 geometry boundary。
- `p_geo_t` 只用于训练辅助，不直接作为最终语义标签。

### 7.3 双反事实分支

构造：

```text
p_geo_depth_cf = G(x_t, aug_d(d_t))
p_geo_rgb_cf = G(aug_x(x_t), d_t)
```

得到：

```text
S_depth = sim(p_geo_t, p_geo_depth_cf)
S_rgb = sim(p_geo_t, p_geo_rgb_cf)
```

### 7.4 时序几何稳定性

对邻近帧：

```text
S_temp = sim(B_geo_t, B_geo_{t-1/t+1})
```

也可以用 patch-level boundary histogram 或 low-resolution geometry map。

### 7.5 几何仲裁权重

```text
A_geo = S_depth * S_rgb * S_temp * exp(-D_rgb_geo)
```

其中：

```text
D_rgb_geo = disagreement(p_rgb_t, p_geo_t)
```

`A_geo` 只控制监督强度，不作为显式可信度目标。

## 8. Loss 设计

### 8.1 Semantic Pseudo-label Loss

语义 hard pseudo-label 来自 RGB teacher：

```text
y_rgb = argmax(p_rgb_teacher)
```

depth 不直接改 `y_rgb`。

```text
L_pl = W_sem * CE(p_student, y_rgb)
```

其中：

```text
W_sem = f(conf_rgb, A_geo)
```

如果 geometry 不稳定，降低 hard pseudo-label 权重。

### 8.2 Boundary Consistency

只在 `A_geo` 高的区域使用 geometry boundary：

```text
L_bd = A_geo * |grad(p_student) - B_geo|
```

### 8.3 Shape / Region Consistency

对稳定 geometry region 做 soft consistency：

```text
L_shape = A_geo * MSE(region(p_student), region(p_geo))
```

### 8.4 Temporal Consistency

对相邻帧结构稳定区域：

```text
L_temp = A_geo * consistency(p_student_t, p_student_{t-1/t+1})
```

### 8.5 总损失

```text
L = L_sup
  + lambda_pl * L_pl
  + lambda_bd * L_bd
  + lambda_shape * L_shape
  + lambda_temp * L_temp
  + lambda_geo_sup * L_geo_sup
```

关键点：

- depth 不直接改语义。
- depth 只做结构监督和权重仲裁。
- 仲裁直接作用于 unlabeled loss。

## 9. 必须实验

### 9.1 主对比

必须包含：

- RGB Mean Teacher
- UniMatch
- U2PL
- SegMatch
- RDNet-style depth co-training
- RGB-D concat
- DepthGuiderV4
- Ours without temporal validation
- Ours without RGB/depth counterfactual
- Ours full

### 9.2 Clean / Corrupted Depth

depth corruption：

- clean
- blur
- noise
- edge drop
- patch shuffle
- random
- wrong depth estimator

目标：

- clean 下优于 RGB-only baseline。
- corrupted 下优于 RDNet/DepthGuider/RGB-D concat。
- random depth 下接近 RGB baseline。

### 9.3 RGB/Depth 双反事实实验

实验：

- 只扰动 RGB。
- 只扰动 depth。
- 同时扰动 RGB 和 depth。
- 比较 arbitration weight 和性能变化。

目的：

- 证明不是普通 augmentation robustness。
- 证明 geometry 使用经过因果验证。

### 9.4 Temporal Jitter Audit

实验：

- 邻近帧 depth boundary stability。
- depth temporal jitter 与 pseudo-label error 的关系。
- 加/不加 temporal validation 的性能和边界稳定性。

这是 TMI 高分关键，因为它体现手术视频特性。

### 9.5 Geometry only affects structure

消融：

- depth 直接更新 semantic label。
- depth 只控制 boundary/shape。
- depth 只控制 pseudo-label weight。

预期：

- depth 只做结构监督更稳。
- 直接更新语义在坏 depth 下更差。

### 9.6 Pseudo-label Error Audit

统计：

- 高 `A_geo` 区域 pseudo-label error。
- 低 `A_geo` 区域 pseudo-label error。
- 被降权区域是否确实错误更多。
- boundary band 内错误是否下降。

### 9.7 RGB-only Inference

必须报告：

- test RGB-only。
- test RGB+depth upper bound。
- train/test depth dependency analysis。

## 10. 论文贡献版本

建议贡献写成：

1. We identify harmful pseudo-depth guidance as an under-studied failure mode in RGB-only semi-supervised surgical instrument segmentation.
2. We propose temporal-causal geometry arbitration, which validates depth-derived structural evidence through RGB/depth counterfactuals and temporal consistency before allowing it to influence unlabeled supervision.
3. We constrain pseudo-depth to provide structural supervision rather than semantic label correction, reducing erroneous geometry-induced confirmation bias.
4. The final model performs RGB-only inference, using pseudo-depth only as privileged training geometry.
5. We provide depth corruption, counterfactual, temporal jitter, and pseudo-label error audits to demonstrate that the method benefits from useful geometry while suppressing harmful geometry.

## 12. 最终建议

如果目标是 TMI 高分，不建议停留在“深度可信度”或“反事实 depth stability”。

应该升级为：

> 时序-因果几何仲裁：用 RGB/depth 双反事实和手术视频时序一致性验证 pseudo-depth 是否能提供稳定结构监督，只让通过验证的几何影响 unlabeled boundary/shape supervision，最终训练 RGB-only student。
