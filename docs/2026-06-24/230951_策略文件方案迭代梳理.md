# 策略文件方案迭代梳理

整理目的：把 `strategies/` 里的策略文件按“自研方案迭代、外部方法对比、内部诊断实验、基础训练策略”拆开，避免只看 `way` 或文件名时混在一起。修改时间只作为辅助证据；同一时间批量出现的文件更像迁移/集中接入，不等价于真实想法产生顺序。

## 总体判断

| 类别 | 代表文件 / way | 判断 |
|---|---|---|
| 当前最相关自研主线 | `semi_georisk_spc.py` / `georisk_spc`, `georisk_spc_dgv4`；`semi_mt_depth_guider_gan_v2.py` / `mt_depth_guider_gan_v2` | 分别对应“深度几何风险/可靠性”和“深度结构先验约束”的两条线，但都还没有形成足够强的 TMI 主叙事闭环。 |
| 深度注入结构迭代 | `semi_mt_depth_guider_v1/v2/v3.py`，以及 `specs.py` 中的 `mt_depth_guider_v4` | 更像模型结构消融：怎么把 depth-derived geometry 注入 RGB 分割网络。不能单独作为论文主贡献。 |
| 语义结构/原型迭代 | `semi_proto_v1.py`，`semi_mt_depth_guider_proto*.py` | 尝试用 prototype/teacher 约束语义空间，但路线复杂，和“深度可信度”主问题连接不够直接。 |
| 对抗几何结构迭代 | `fully_supervised_depthGAN.py`，`semi_mt_depth_guider_gan_v1/v2/v3.py` | 用 label-depth / pred-depth 对抗约束结构合理性。可作为补充模块或负例分析，不建议作为唯一主线。 |
| 外部方法对比 | `semi_rdnet.py`, `semi_unimatch*.py`, `semi_segmatch*.py`, `semi_u2pl.py`, `semi_corrmatch.py`, `semi_cps.py`, `semi_mms.py`, `semi_cwbass.py` 等 | 用于证明你的方法不是普通 SSL trick 或普通多模态可靠性变体。RDNet 是最关键相关对比。 |
| 内部诊断/消融 | `only_depth_input`, `fully_rgb_masking_depth_v1`, `fully_depth_pretrain_v1`, `fully_contrast_v1*` | 用于回答“depth 是否有用、怎么用、坏 depth 会怎样、边界/结构是否受益”。 |

## 当前最清晰的迭代关系

| 迭代线 | 顺序 | 应该怎么讲 |
|---|---|---|
| DepthGuider | `v1 -> v1_2 -> v2 -> v3 -> v4` | 这是“怎么注入 depth geometry”的模型结构探索；主论文中最多保留最强版本和关键消融。 |
| GeoRisk | `georisk_spc -> georisk_spc_dgv4 -> georisk_spc_dgv4_v2` | 这是“如何处理 depth 不可信”的可靠性路线；但 v2 当前实现退化为 teacher uncertainty，不应当代表最终方法。 |
| DepthGAN | `fully_depthgan -> mt_depth_guider_gan_v1 -> v2 -> v3` | 这是“depth-aware shape prior 是否能约束 segmentation”的结构先验路线；v1 是失败暴露，v2/v3 是风险控制版本。 |
| Prototype | `proto_v1 -> depth_guider_proto_v1 -> proto_teacher_v1/v2/v3` | 这是“语义空间结构化”的探索线；目前不建议和 GeoRisk/GAN 同时作为主贡献，否则显得堆模块。 |
| 外部 SSL 对比 | `MT/UAMT/URPC/CPS -> UniMatch/U2PL/CorrMatch/SegMatch/RDNet` | 这是对比矩阵，不是你的方案迭代。RDNet、U2PL、UAMT、CWBASS 与可靠性叙事最相关。 |

## 建议保留的论文实验矩阵

| 目的 | 推荐策略 | 不推荐做法 |
|---|---|---|
| 证明 depth 有信息但不能全信 | `only_depth_input`, `fully_rgb_masking_depth_v1`, `mt_depth_guider_v4` | 只报“拼 depth 提升 Dice”，说服力弱。 |
| 证明几何可靠性必要 | `georisk_spc` vs `uamt` vs `u2pl` vs `cwbass` | 只和 `mt` 比，会被认为是普通置信度筛选。 |
| 证明训练时 depth 辅助、推理时 RGB 的价值 | `rdnet` 必须对比，最好补 RGB-only 推理协议 | 如果当前方法测试仍依赖 depth，就不能写成纯 RGB 推理。 |
| 证明坏 depth 不会误导 | clean depth / corrupted depth / shuffled depth / RGB-only 的一致实验 | 只给正常 depth 结果，无法支持“可信度/可靠性”论点。 |
| 证明边界几何收益 | boundary Dice/NSD/HD95、thin-structure region、depth discontinuity band | 只报平均 Dice，无法体现几何信息价值。 |

## 一句话整理

当前代码库不是一条线，而是四条线并行：`DepthGuider` 解决“怎么用 depth”，`GeoRisk` 解决“坏 depth 怎么抑制”，`DepthGAN` 解决“几何结构先验怎么施加”，`Prototype/Text/Contrast` 是语义结构旁路；外部 baseline 则用于证明这些不是普通 SSL 置信度筛选或多模态互教。若目标是 TMI 主线，建议以 `GeoRisk + DepthGuiderV4 + RDNet/UAMT/U2PL/CWBASS 对比 + corrupted depth 审计` 收束，不要把所有策略都塞进一个方法。
