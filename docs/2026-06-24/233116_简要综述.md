# 简要综述

时间基准：`2026-06-24`

## 1. 半监督手术器械与内镜分割

到 2025-2026，这条线已经不再停留在 `Mean Teacher + consistency`。`SegMatch` 把 `FixMatch` 式强弱增强、伪标签和对抗增强带到 laparoscopic / robotic surgical segmentation，证明手术器械分割已经有了专门的 semi-supervised 方案。进入 2026，`Endo-SemiS` 进一步把问题推到内镜视频场景，强调 uncertainty-guided pseudo-label、dual-network cross-supervision、joint pseudo-label supervision 和 spatiotemporal corrective learning，说明审稿人已经默认你需要解释手术视频中的噪声、时序和伪标签失真，而不仅仅是“标注少所以做 SSL”。

这意味着你的工作不能只把自己放在“手术器械半监督分割”这一层。真正的竞争点已经变成：在手术场景中，哪些未标注像素能被强监督，哪些像素必须被弱监督或延迟使用。

## 2. 医学 SSL 中的伪标签可靠性主线

`UAMT`、`URPC` 这类医学 SSL 方法已经把 uncertainty 引入 consistency 学习，用预测不确定性抑制错误监督。自然图像侧，`CPS` 给出双网络互教，`U2PL` 明确提出 unreliable pseudo-label 不应简单丢弃，而应被转化利用；`UniMatch` 和 `CorrMatch` 则把 weak-to-strong consistency、correlation propagation、阈值选择做成强基线。到 `ICCV 2025` 的 `When Confidence Fails`，文献已经开始直接质疑 confidence-based pseudo-label selection 的分离能力，说明“高置信错误”“阈值不可靠”已经是共识问题。边界感知方向上，`CW-BASS (IJCNN 2025)` 又进一步说明，仅凭 confidence 做 boundary-aware weighting 已经是一条独立成熟路线。

因此，单独提出 `entropy/confidence 不够` 已没有新意。你如果只做 `teacher uncertainty + consistency`，会自然落回现有可靠性分流路线。要成立新的贡献，必须证明你的筛选依据不是再造一个 confidence rule，而是引入了现有方法没有的结构证据。

## 3. 单目深度与伪深度在内镜场景中的角色

`Depth Anything V2`、`Metric3Dv2` 这类 foundation depth model 让外部 pseudo-depth 更容易获得，也让“训练时额外拿到一张深度图”变得现实。但这并不等于 pseudo-depth 在手术场景中天然可靠。相反，手术图像里的反光、烟雾、血液、快速视角变化、器械遮挡和细长结构，本身就会导致深度边界断裂、局部错位和时序抖动。`EndoStreamDepth (MIDL 2026)` 的出现本身就说明：在 endoscopic video 中，深度的一大核心问题不是“能不能估出来”，而是“能否时间上稳定且边界上锐利”。

这对你的方案有两个直接启发。第一，pseudo-depth 更适合作为 `局部几何先验`，而不是直接当作第二语义模态。第二，几何证据必须是可筛选的，因为同一张 depth 图里既有可用结构，也有会污染学习的错误边界。

## 4. RGB-D / pseudo-depth 分割中的已有共识

`DFormer`、`DFormerv2`、`PrimKD` 这类 RGB-D 文献已经提供了两层重要共识。第一，depth 可以被视作 `geometry prior`，不一定要与 RGB 完全对称建模。第二，RGB 往往仍是主模态，depth 更适合作为从属或约束信号，而不是直接接管语义判断。这与你当前方案想强调的“depth 只约束结构，不直接改类别”是对齐的。

但这些工作大多属于监督 RGB-D segmentation：测试阶段继续输入 depth，重点是融合而非风险控制。它们并没有回答半监督场景里一个更尖锐的问题：当 depth 本身来自 foundation model 且局部错误不可避免时，是否应让它直接参与伪标签更新，还是先审计其几何证据再决定是否使用。

`RDNet (CVPR 2025)` 是最接近你的工作，它已经完成了 `train-depth / test-RGB`、depth-guided mutual learning 和 depth-guided hard patch augmentation。也正因如此，你的切入点不能再是“让 depth 帮助 RGB”，而必须是“限制坏 depth 误导 RGB”。

进一步看，多模态可靠融合文献已经把 `conflict-guided evidential fusion`、`sensor degradation under reliability-aware fusion` 做成成熟方向。这些工作证明“模态并不总可靠”早已是已知问题，但它们默认测试期仍可访问多模态输入。你的设定必须明确区分为：`训练期有 noisy pseudo-depth，测试期只有 RGB`。否则论文会被吸进通用 unreliable modality fusion 文献池。

## 5. foundation model 与伪标签修正的新压力

2025-2026 之后，`SemiSAM+`、`SSL-MedSAM2`、foundation-model guided iterative prompting / pseudo-labeling 等工作已经把“用大模型帮助半监督医学分割”推成显性方向。与此同时，`SEMI-PLC`、`MCF`、conflict-aware mutual learning、spatially decoupled reliable mutual learning 等工作继续把 fragile prediction identification、pseudo-label correction、mutual correction 和 conflict utilization 做细。

这说明如果你的叙事只是“外部模型提供更好的先验，帮助修正伪标签”，审稿人很容易把它归到 foundation-model pseudo-label refinement 或 conflict-aware correction 一类。你的差异只能落在：这些工作主要修正 `语义伪标签`，而你研究的是 `几何伪先验` 如何在训练期被选择性利用和抑制。

## 6. 你的切入点应放在哪里

结合 2026 的文献状态，最有区分度的落点不是 `depth-aware segmentation`，也不是 `pseudo-label reliability`，而是二者的交集：

> `foundation-model 生成的 pseudo-depth 在手术场景中同时具有帮助性与误导性，因此它不应被直接并入伪标签学习，而应先被验证为局部结构证据，再参与未标注监督仲裁。`

这个切入点有三层边界必须保持清楚。

第一，语义类别仍由 RGB teacher/student 决定，depth 不直接改 label。第二，depth 的作用应限制在结构层面，例如边界、连通性、局部一致性和监督权重。第三，若进一步加入反事实或时序模块，其目的应是验证 geometry 是否稳定，而不是把工作改写成完整的视频时序分割框架。

按这个口径组织 related work，会形成很清晰的差异链：

- `SegMatch / Endo-SemiS` 解决 surgical SSL 和视频鲁棒性
- `UAMT / URPC / U2PL / CorrMatch / CW-BASS / When Confidence Fails` 解决伪标签可靠性与边界感知筛选
- `DFormer / DFormerv2 / PrimKD` 说明 depth 可作 geometry prior 且 RGB 应保持主导
- `RDNet` 证明 train-depth/test-RGB 已可行，但默认 depth guidance 有益
- 你的工作进一步提出：`pseudo-depth 需要先被仲裁，再能调节未标注结构监督`

一句话总结：

`到 2026，这条线真正还能讲的新问题，不是“如何使用 depth”，而是“如何在 pseudo-depth 不稳定的前提下，只保留它对半监督结构学习真正有益的部分”。`
