# RDNet复现审议报告

## 结论

当前 `strategies/semi_rdnet.py` 不是原 RDNet 的等价复现，且存在一个进入 DPA 分支后会直接报错的问题。主要偏差集中在：DPA 输入、强增强 `image_s`、二分类 sigmoid/BCE-Dice 与本项目 softmax/CE-Dice 的语义不一致、特征一致性使用 logits 代替 encoder feature、优化器与原实现不一致。

## 最高优先级问题

1. `strategies/semi_rdnet.py:238` 使用 `self.args.max_epoch`，但本项目参数没有 `max_epoch`。只要 `iter_num >= consistency_start_iters` 且 batch 中存在 `depth1`，DPA 分支会触发 `AttributeError`。

2. `strategies/semi_rdnet.py:238` DPA 使用 `unlabeled_image`，原 RDNet 使用强增强图 `unlabeled_img_s`。原仓库流程是 `images_s -> unlabeled_img_s -> dpa(...) -> model(unlabeled_img_s_cutmix)`，当前项目没有真正的 `image_s` 数据流。

3. `data/transforms.py:124` 当前 `RandomGenerator` 不生成 RDNet 需要的 `image_s`。原仓库在 `datasets/kvasir_SEG.py` 中对 `image_s` 做 `ColorJitter(0.5, 0.5, 0.5, 0.25)` 和 `GaussianBlur`，当前只做旋转/翻转，再在策略里加 Gaussian noise，语义不一致。

4. `strategies/semi_rdnet.py:194` 当前使用 `softmax + CE/Dice` 多类范式，原 RDNet 是 `sigmoid + BCE-Dice` 二分类范式。若本项目二分类以 `num_classes=2` 跑，`max softmax >= 0.85` 与原来的 `pred > 0.85 or pred < 0.15` 不是同一套伪标签筛选逻辑。

5. `strategies/semi_rdnet.py:230` `update_pseudo_labels` 直接套到多通道 softmax 上。原函数假设单通道概率图，用 `>0.5` 和 `<1-gamma` 处理前景/背景置信度；多类 softmax 下逐通道套用会破坏概率分布。

## 训练信号偏差

1. `strategies/semi_rdnet.py:218` 当前把无监督、DPA、对比、MSE、特征一致性全部延迟到 `consistency_start_iters=1000` 后。原 RDNet 从第 0 iter 开始计算这些 loss。

2. `strategies/semi_rdnet.py:257` 特征 L2 用 `rgb_logits/depth_logits` 近似。原 RDNet 用 `model(..., fp=True)` 返回的深层特征 `e5_u/e5_u_d`，这不是同一个约束。

3. `strategies/semi_rdnet.py:271` `loss_rgb` 使用 `self.rdnet_thresh_depth * cross_cons`。原公式是 `0.5 * loss_u_rgb_d`，类里已有 `rdnet_cross_weight=0.5` 但没有使用。

4. `strategies/semi_rdnet.py:224` 当前无监督损失是 masked MSE。原 RDNet 的 `BceDiceLoss1` 是 masked BCE + Dice，监督和无监督 loss 语义都不同。

5. `core/train.py:154` 本项目主优化器是 Adam，`strategies/semi_rdnet.py:54` 深度分支也是 Adam；原 RDNet 是 `SGD(momentum=0.9, weight_decay=1e-5)`。

## 数据与输入风险

1. RDNet 需要同时加载 `depth3/depth1`。本项目只有 `--use_depth 13` 才会同时加载两者。若只传 `--use_depth 3`，DPA 没有 `depth1`；若只传 `--use_depth 1`，depth 分支输入会由 1 通道重复成 3 通道，不是原来的 `depth_rgb`。

2. `data/transforms.py:160` 当前旋转范围是 `[-20, 20]`，原 RDNet 是 `RandomRotation(90)`，且原实现还有 `RandomZoom(0.9, 1.1)`。

3. `data/dataset.py:233` 图像默认归一化按本项目参数走，depth 只按 dtype 缩放到 `[0,1]`。原 RDNet 使用 `torchvision.transforms.ToTensor()`，图像、depth、label 都是单纯除以 255，没有 ImageNet 标准化。

## 原实现关键对照

原 RDNet 主训练链路：

```python
ema_pred_u = ema_model(unlabeled_img)
pred, e5 = model(torch.cat((unlabeled_img_s, labeled_img)), fp=True)
pred_d, e5_d = model_depth(torch.cat((unlabeled_depth, labeled_depth)), fp=True)
unlabeled_img_s_cutmix, ema_pred_u_cutmix = dpa(unlabeled_depth1, unlabeled_img_s, ema_pred_u, beta=0.3, t=epoch, T=args.nEpoch)
pred_u_cutmix = model(unlabeled_img_s_cutmix)
loss_supervised_rgb = criterion(pred_l, gts, threshold=-1)
loss_u_rgb = criterion(pred_u, ema_pred_u, threshold=args.thresh)
loss_u_rgb_d = criterion_d(pred_u, pred_u_up, threshold=args.thresh_depth)
loss_u_rgb_dpa = criterion(pred_u_cutmix, ema_pred_u_cutmix, threshold=args.thresh_dpa)
loss_rgb = (loss_supervised_rgb + 0.5 * loss_u_rgb + loss_u_rgb_dpa + 0.5 * loss_u_rgb_d) / 3.0
loss_depth = (loss_supervised_depth + 1.2 * loss_u_depth) / 2.0
loss_con = (loss_cont + 0.5 * loss_mse + 0.5 * loss_con_f) / 2.0
loss_total = (loss_rgb + loss_depth + loss_con) / 3.0
```

当前项目对应实现最大差异：

```python
rgb_logits = self.model(rgb_input)
depth_logits = self.depth_model(depth_input)
rgb_prob = torch.softmax(rgb_logits, dim=1)
depth_prob = torch.softmax(depth_logits, dim=1)
unsup_rgb = self._compute_masked_consistency(rgb_u, ema_prob, conf_mask)
dpa_input, dpa_teacher = self._dpa_augment(unlabeled_depth1, unlabeled_image, ema_prob, epoch, max(1, self.args.max_epoch))
loss_feat_l2 = feature_l2_loss(rgb_logits[labeled_bs:], depth_logits[labeled_bs:].detach())
loss_rgb = (sup_rgb + 0.5 * unsup_rgb + dpa_cons + self.rdnet_thresh_depth * cross_cons) / 3.0
```

## 建议修复顺序

1. 先修 `self.args.max_epoch`，保证进入 DPA 后不崩。
2. RDNet 运行入口强制使用 `--use_depth 13`，否则不是完整 RDNet。
3. 给 RDNet 单独数据增强，生成 `image_s`，DPA 使用 `image_s`。
4. 明确复现目标：若严格复现原 RDNet，应改回 sigmoid/BCE-Dice 二分类语义；若保留本项目多类 softmax，则只能称为 RDNet-inspired 适配版。
5. 用模型真实 encoder feature 替代 logits 做 L2，或删除“特征一致性”声明。
6. 将 `cross_cons` 权重改回 0.5，并确认无监督 loss 是否改成 masked BCE-Dice。
7. 对齐优化器和学习率：原 RDNet 使用 SGD、`lr=1e-3`、`weight_decay=1e-5`、poly scheduler。
