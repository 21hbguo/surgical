# 隐式几何可信度路线评审

时间：2026-06-24 21:33:12

## 1. 核心判断

“深度几何可信度”这个问题是有价值的，但不建议把方法写成显式可信度建模。

原因：

- 显式 `trust map = entropy + depth edge + conflict` 容易显得手工、低级、像 heuristic。
- 固定阈值分高低可信区域很难调，消融容易非单调。
- 可信度本身没有直接 ground truth，评估会变成间接解释，审稿人容易质疑“这个 map 到底是不是可信度”。
- 2025-2026 已经有很多 uncertainty-guided pseudo-label、confidence filtering、pseudo-label reliability 方法，单纯显式可信度很难形成强 novelty。

更好的定位不是“估计深度可信度”，而是：

> 学习在半监督分割训练中什么时候利用几何，什么时候抑制几何。

建议把表述从 `depth reliability estimation` 改成：

**implicit geometry arbitration**

或者：

**task-driven geometry arbitration**

核心区别：

- 不直接声明预测了一个可信度真值。
- 不强调手工公式是否准确。
- 强调模型通过分割目标和伪标签风险自动学习几何使用策略。
- 评估重点从“可信度图准不准”转向“坏 depth 下是否少受伤，好 depth 下是否能提升”。

## 2. 为什么显式可信度不够强

显式可信度最大的问题是它本身难以定义。

对单目深度而言，“可信”至少有三层含义：

1. 物理深度是否准确。
2. 深度边界是否对应真实物体边界。
3. 深度信息是否对当前分割任务有帮助。

医学/手术器械分割里，第三点最关键。单目深度的绝对值可以不准，但只要局部边界、遮挡关系、器械-组织结构对分割有帮助，它就是有用的。反过来，深度图看起来平滑、稳定，也不一定对器械边界有帮助。

因此，如果显式定义 `Q_d`，审稿人会问：

- `Q_d` 是 depth accuracy 吗？
- 是 pseudo-label reliability 吗？
- 是 boundary reliability 吗？
- 是 segmentation usefulness 吗？

这些概念混在一起，会削弱论文说服力。

## 3. 更推荐的隐式方案

### 3.1 方案 A：Implicit Geometry Arbitration

设计 RGB expert 和 geometry expert，中间用一个轻量 arbitration gate 融合。

输入：

- RGB image
- monocular pseudo-depth，只在训练或辅助分支使用

核心：

- RGB branch 负责主分割。
- geometry branch 从 depth gradient / depth feature 中提取几何提示。
- arbitration gate 根据 RGB feature、geometry feature、teacher uncertainty、prediction disagreement 自动决定几何残差注入强度。

关键是不要把 gate 命名为 trust map。可以叫：

- geometry arbitration weight
- geometry residual gate
- geometry usage coefficient

loss 里不直接监督 gate，而是通过 segmentation loss、consistency loss、pseudo-label correction 间接学习。

优点：

- 比手工可信度高级。
- 可解释但不强依赖解释。
- 能通过 depth corruption 证明 gate 有用。
- 可以保留 DepthGuiderV4 的残差注入思想，但改成更任务驱动。

### 3.2 方案 B：Task-Driven Geometry Prompt

不要让模型学习真实 depth，也不要显式判断 depth 是否可信，而是学习一个对分割有帮助的 geometry prompt。

思路：

- depth estimator 提供 pseudo-depth。
- 网络从 pseudo-depth 中提取 geometry prompt。
- prompt 不是为了还原 depth，而是为了改善 segmentation boundary 和 pseudo-label training。
- 如果 depth 缺失或退化，prompt 分支通过训练自动减弱。

这个方向比“深度可信度”更容易写得高级：

> Monocular depth is not used as a target or modality to be trusted, but as a noisy geometric prompt whose task utility is learned through segmentation supervision.

### 3.3 方案 C：Depth as Privileged Training Signal

训练时使用 depth，测试时只使用 RGB。

具体做法：

- teacher 或 auxiliary branch 可以看 RGB+depth。
- student 只看 RGB。
- depth 通过 feature distillation、boundary distillation、pseudo-label selection 或 geometry consistency 影响 student。
- 测试阶段完全不需要 depth。

这条路线很适合“为纯 RGB 图像增加几何信息”这个目标。

论文表述可以是：

> We use monocular depth as privileged geometry during semi-supervised training and distill geometry-aware supervision into an RGB-only student.

这样比“RGB-D segmentation”更有现实意义。

## 4. 推荐最终主线

建议主线定为：

**Implicit Geometry Arbitration for RGB-only Semi-supervised Surgical Instrument Segmentation**

可以拆成三个模块：

1. Geometry Prompt Branch：从 monocular depth 里提取 task-driven geometry prompt。
2. Arbitration Fusion：用隐式 gate 控制 geometry residual 对 RGB feature 的影响。
3. Geometry-aware Unlabeled Supervision：用 teacher-student consistency 和 pseudo-label disagreement 驱动几何提示学习。

不要主打“可信度图”。

可以在 discussion 或 analysis 里说：

> The arbitration weight can be interpreted as geometry usage rather than calibrated depth reliability.

这样避免被审稿人抓住“可信度没有 ground truth”。

## 5. 如何评估隐式方法

隐式方法不是不评估，而是评估因果效果，不评估 map 本身是否像可信度。

### 5.1 Depth Corruption Curve

这是最关键的实验。

设置：

- clean pseudo-depth
- Gaussian noise depth
- blurred depth
- shuffled depth
- random depth
- edge-eroded depth
- wrong estimator depth

对比：

- RGB baseline
- RGB + depth concat
- DepthGuiderV4
- explicit trust map
- implicit geometry arbitration

理想结果：

- clean depth 下，implicit method 高于 RGB baseline。
- corrupted depth 下，implicit method 下降小于 RGB-D concat 和 DepthGuiderV4。
- random depth 下，implicit method 接近 RGB baseline，而不是崩溃。

这个实验能直接证明：模型不是盲信 depth。

### 5.2 RGB-only Inference

如果目标是“为纯 RGB 图像增加几何信息”，必须做测试只输入 RGB。

对比：

- RGB-only supervised/semi-supervised baseline
- train RGB+depth, test RGB
- train RGB+depth, test RGB+depth
- proposed train-depth/test-RGB

结论要落在：

> Depth is used as training-time geometric supervision, not as a deployment-time requirement.

### 5.3 Good-depth / Bad-depth Intervention

同一张 RGB 图像配不同 depth：

- original depth
- high-quality depth
- noisy depth
- shuffled depth
- random depth

观察 prediction 是否稳定。

好方法应该：

- 好 depth 改善边界。
- 坏 depth 不显著破坏预测。
- random depth 下不比 RGB baseline 差太多。

### 5.4 Gate Usage Analysis

如果方法有 gate，可以做辅助分析：

- gate 在 instrument boundary 区域是否更活跃。
- gate 在 depth corruption 下是否自动降低。
- gate 与 teacher disagreement 是否相关。
- gate 与 pseudo-label error 是否有统计相关。

注意：这只是辅助解释，不要把 gate 当 ground-truth trust map。

### 5.5 Pseudo-label Error Audit

在 held-out labeled data 上做 post-hoc audit：

- 按 arbitration weight 分桶。
- 统计每个桶的 pseudo-label error。
- 统计 Boundary F1、HD95、Dice。
- 统计 high-gradient band 内的错误率。

这样可以说明 arbitration 确实和伪标签风险相关，但不需要声明它就是可信度真值。

## 6. 和现有代码的关系

当前可以保留的部分：

- DepthGuiderV4 的 residual depth injection 思路。
- GeoRisk-SPC 里 depth discontinuity、teacher uncertainty、geometry-semantic conflict 的诊断变量。
- Mean Teacher 训练主线。
- risk-guided perturbation 可以改成 arbitration-guided perturbation。

建议放弃或降级：

- `mt_depth_guider_gan_v1` 不适合作为主线。
- `georisk_spc_dgv4_v2` 当前不适合作为可信度方法，因为 depth 没有进入 risk 计算。
- 固定阈值 `tau_r/tau_c` 不适合作为核心贡献。
- 显式 `M_r/M_l` 高低风险硬划分要谨慎，最好改成连续权重或软路由。

建议新增策略名：

- `geo_arbitration_rgb_v1`
- `implicit_geo_rgb_v1`
- `geometry_prompt_mt_v1`

避免继续在 `georisk_spc_dgv4_v2` 上堆概念。

## 7. 推荐实验命令方向

保留当前训练入口，但新策略应该满足：

- 训练时 `--use_depth 1`
- validation/test 可以选择 RGB-only
- loss 日志里记录 geometry usage，而不是 trust accuracy

示例方向：

```bash
CUDA_VISIBLE_DEVICES=0 python -m core.train --fold -1 --task 1 --way implicit_geo_rgb_v1 --exp endovis2017_h5_224_224/ImplicitGeoRGB_v1 --labeled_num 40 --use_depth 1
```

后续要配套 corruption eval，而不是只跑 clean depth。

## 8. 推荐论文贡献表述

建议写：

1. We revisit monocular-depth-assisted semi-supervised surgical instrument segmentation and identify that direct depth fusion is vulnerable to unreliable pseudo-depth.
2. We propose an implicit geometry arbitration framework that learns when to inject or suppress depth-derived geometry without requiring calibrated depth reliability labels.
3. We use monocular depth as privileged training geometry and distill its task-relevant cues into an RGB-only student.
4. We validate the method through depth corruption, RGB-only inference, and pseudo-label error audits, showing robustness to unreliable depth.

不要写：

- We estimate accurate depth confidence.
- We compute a reliable trust map.
- Low-risk and high-risk masks are correct by design.

这些表述都容易被攻击。

## 9. 最终建议

“可信度”作为问题动机是好的；“显式可信度建模”作为方法核心不够好。

最好的改法是把它升维为：

> 隐式几何仲裁：模型不判断 depth 是否真实可信，而是学习 depth-derived geometry 对当前分割监督是否有用。

这个角度更高级，也更容易评估。评估重点不是 map 的准确率，而是：

- 好 depth 是否带来收益。
- 坏 depth 是否被抑制。
- 测试 RGB-only 是否仍能受益。
- 伪标签错误区域是否被更稳定地处理。
